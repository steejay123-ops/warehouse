# گزارش پیاده‌سازی سیستم مدیریت کاربران و سطح دسترسی (RBAC)

توسعه و انتقال سیستم مدیریت کاربران، نقش‌ها و سطوح دسترسی از حالت Mock به API واقعی با موفقیت انجام شد.

## کارهای انجام شده در Backend
- فیلد `role` از مدل `CustomUser` حذف و به جای آن از سیستم `Group` و `Permission` پیش‌فرض جنگو استفاده شد.
- فیلد `assigned_warehouses` به عنوان یک فیلد چندبه‌چند (ManyToManyField) برای تخصیص انبارها به کاربران ایجاد شد.
- دسترسی‌های سیستمی مورد نیاز فرانت‌اند (مثل `perm_wh_create`، `perm_sys_settings` و...) طی یک اسکریپت در دیتابیس `auth_permission` ثبت شدند.
- در سیستم احراز هویت (`UserSerializer`)، کاربرانی که نقش `admin` دارند یا `is_superuser` هستند به صورت خودکار به تمام انبارها دسترسی خواهند داشت.

## کارهای انجام شده در Frontend
- سرویس `AccountsHttpService` برای مدیریت ارتباط با API های کاربران، نقش‌ها و دسترسی‌ها ایجاد شد.
- کامپوننت و فایل HTML مربوط به `users` به روزرسانی شدند تا به جای استفاده از داده‌های استاتیک در `state.service.ts`، از API برای دریافت و ارسال اطلاعات استفاده کنند.
- فرم‌های تعریف کاربر جدید، تخصیص نقش، تعیین انقضای رمز عبور و مسدودسازی کاربر مستقیما به سیستم Backend متصل شدند.

## تکمیل یکپارچه‌سازی مدل انبار (Warehouse)
- **به‌روزرسانی مدل دیتابیس:** اضافه کردن فیلدهای جدید (کد، پروژه، فعال/غیرفعال) به جای `status` قدیمی. تمامی مایگریشن‌های اضافی حذف و از نو ایجاد شد تا تداخلی پیش نیاید.
- **اصلاح API سمت سرور:** به‌روزرسانی `WarehouseViewSet` برای استفاده از `is_active` به جای فیلد وضعیت رشته‌ای و مدیریت صحیح فریز.
- **هماهنگ‌سازی پنل فرانت‌اند:** کدهای `projects.ts` و فایل HTML مربوط به آن (`projects.html`) کاملاً بازنویسی شدند تا به جای کدهای دکوری، مستقیماً به API اصلی بک‌اند متصل شوند. اکنون دکمه **"انبار جدید"** و **"ویرایش"** مقادیر را مستقیماً در دیتابیس ذخیره می‌کنند و از فیلد `is_active` پشتیبانی می‌کنند.

## راهنمای اجرای مجدد برنامه
برای سوال شما در مورد راه‌اندازی مجدد برنامه در روزهای آینده:
فقط کافیست روی فایل `start.bat` که در روت پروژه (کنار پوشه‌های بک‌اند و فرانت‌اند) قرار دارد دو بار کلیک کنید.
این فایل به صورت خودکار دو صفحه سیاه (CMD) برای بک‌اند و فرانت‌اند باز می‌کند و پروژه را در آدرس `http://localhost:4200` در دسترس قرار می‌دهد. اگر با خطای پورت مواجه شدید، مطمئن شوید پنجره‌های سیاه (به خصوص NodeJS) قبلی را بسته‌اید.

## پاسخ به سوالات شما

> می خواهم وقتی برنامه را می بندم و دوباره برمیگردم برنامه را اجرا کنم چه باید بکنم؟

برای اجرای مجدد برنامه، کافیست به پوشه اصلی پروژه `warehouse project` بروید و فایل **`start.bat`** را اجرا کنید (دابل‌کلیک کنید). این فایل به صورت خودکار دو پنجره باز می‌کند: یکی برای سرور جنگو (Backend) و دیگری برای سرور انگولار (Frontend) و نیازی به هیچ دستور اضافه‌ای نخواهید داشت.

> آیا جدول کاربران و جدول های وابسته ، مثل اینکه هر کاربر چه نقشی دارد و یا اینکه به چه مواردی دسترسی دارد یا نه، الان تکمیل است؟ 

**بله، کاملاً تکمیل شده است.** 
از این پس:
- نقش‌ها (Roles) در جدول `auth_group` جنگو ذخیره می‌شوند.
- دسترسی‌ها (Permissions) در جدول `auth_permission` ذخیره می‌شوند.
- هر نقش مجموعه‌ای از دسترسی‌ها دارد (`group_permissions`).
- هر کاربر به یک یا چند نقش متصل می‌شود (`user_groups`).
- هر کاربر به تعدادی انبار دسترسی مجاز دارد (`assigned_warehouses`).
همه این موارد در دیتابیس به درستی ارتباط (Relation) دارند و با موفقیت Migrate شده‌اند.
