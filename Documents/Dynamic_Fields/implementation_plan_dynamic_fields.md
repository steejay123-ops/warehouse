# برنامه‌ریزی پیاده‌سازی: فیلدهای پویا (Dynamic Fields)

ایجاد قابلیت افزودن، ویرایش و مدیریت فیلدهای دلخواه برای کالاها توسط کارفرما با استفاده از معماری ترکیبی (JSONField).

> [!IMPORTANT]
> این تغییر شامل اضافه شدن یک جدول جدید برای تعاریف فیلدها و یک ستون `JSONField` در جدول کالاها می‌باشد. به دلیل استفاده از معماری ترکیبی، نیازی به ری‌استارت شدن سرور پس از افزودن فیلدها نیست.

## تغییرات پیشنهادی

### بک‌اند (Django)

| فایل | نوع تغییر | توضیحات |
| :--- | :--- | :--- |
| `inventory/models.py` | `[MODIFY]` | اضافه کردن مدل `ItemFieldDefinition` و افزودن `dynamic_data = models.JSONField()` به `Item`. |
| `inventory/serializers.py` | `[MODIFY]` | ساخت سریالایزر برای مدل جدید و آپدیت سریالایزر `Item` جهت پشتیبانی از دیتای متغیر. |
| `inventory/views.py` | `[MODIFY]` | ساخت ViewSet برای مدیریت فیلدهای پویا (CRUD). |
| `inventory/urls.py` | `[MODIFY]` | ثبت روت‌های مربوط به ViewSet فیلدهای پویا. |

### فرانت‌اند (Angular)

| فایل | نوع تغییر | توضیحات |
| :--- | :--- | :--- |
| `src/app/core/models/dynamic-field.model.ts` | `[NEW]` | ایجاد اینترفیس‌های مربوط به تعاریف فیلدهای پویا. |
| `src/app/core/api/dynamic-field-api.service.ts` | `[NEW]` | سرویس ارتباط با API فیلدهای پویا. |
| `src/app/features/settings/dynamic-fields/` | `[NEW]` | کامپوننت‌های مدیریت فیلدها (صفحه ادمین برای اضافه/کم کردن فیلدها). |
| `src/app/features/inventory/item-form/` | `[MODIFY]` | تغییر فرم ثبت و ویرایش کالا جهت رندر کردن داینامیکِ فیلدهای جدید. |

## برنامه‌ی اعتبارسنجی (Verification Plan)

### تست‌های خودکار
- اجرای `python manage.py makemigrations` و `migrate` برای اطمینان از سلامت دیتابیس.
- بیلد گرفتن فرانت‌اند با `npm run build` جهت اطمینان از عدم وجود خطای تایپ‌اسکریپت.

### تست‌های دستی
1. ورود به صفحه مدیریت فیلدها و ساخت فیلد جدید.
2. ورود به فرم کالا و اطمینان از نمایش صحیح فیلدها در فرم.
3. پر کردن فرم کالا، ذخیره و لود مجدد برای اطمینان از ذخیره‌ی موفق اطلاعات.
4. ویرایش نام یکی از فیلدها در پنل مدیریت و اطمینان از اعمال آن در تمامی فرم‌های کالا بدون حذف دیتا.
