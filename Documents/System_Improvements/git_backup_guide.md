# راهنمای بک‌آپ‌گیری سریع با Git

این راهنما شامل دستورات ضروری برای مدیریت کدهای پروژه و جلوگیری از بروز مشکلات (Regression) است. هر زمان که یک بخش از پروژه کامل شد و به درستی کار می‌کرد، پیشنهاد می‌شود یک بک‌آپ بگیرید.

## ۱. مشاهده وضعیت فایل‌ها
ابتدا چک کنید چه فایل‌هایی تغییر کرده‌اند:
```bash
git status
```
*رنگ قرمز به معنای فایل‌هایی است که تغییر کرده‌اند اما هنوز آماده ذخیره نیستند.*

## ۲. اضافه کردن تمام تغییرات (Stage)
برای آماده‌سازی تمامی فایل‌های تغییر یافته جهت بک‌آپ:
```bash
git add .
```

## ۳. ثبت بک‌آپ (Commit)
حالا تغییرات را با یک پیام معنادار ذخیره کنید (پیام را متناسب با کاری که انجام دادید تغییر دهید):
```bash
git commit -m "backup: features worked successfully"
```

---

## مواقع اضطراری: بازگردانی به آخرین نسخه سالم
اگر بخشی از پروژه خراب شد و خواستید تمامی تغییرات ذخیره نشده (Uncommitted) را حذف کنید و به آخرین نسخه سالمی که کامیت کرده بودید برگردید:

> [!CAUTION]
> <div dir="rtl" align="right">این دستور تمام کارهایی که بعد از آخرین بک‌آپ انجام داده‌اید را کاملاً پاک می‌کند و غیرقابل بازگشت است!</div>

```bash
git restore .
```
اگر فایل جدیدی ساخته بودید که می‌خواهید آن هم پاک شود:
```bash
git clean -fd
```

## مشاهده تغییرات قبل از ذخیره
اگر می‌خواهید دقیقاً ببینید در کدها چه خطوطی حذف یا اضافه شده‌اند:
```bash
git diff
```
