# طرح پیاده‌سازی: بهبود فرآیند و توسعه ویژگی‌های درخواستی

با تشکر از بازخورد صریح شما. این طرح برای اجرای دقیق درخواست‌های سه‌گانه شما تنظیم شده است. لطفاً پس از مطالعه، آن را تایید کنید تا اجرا را آغاز کنم.

## ۱. بروزرسانی قوانین `AGENTS.md`
- **هدف:** جلوگیری از انجام هرگونه تغییر بدون تایید صریح شما.
- **تغییرات:** یک قانون جدید به بخش `AGENTS.md` اضافه می‌شود که مرا ملزم می‌کند برای تمامی درخواست‌ها (حتی تغییرات کوچک) از `Planning Mode` استفاده کرده و یک فایل `implementation_plan` برای تایید شما بسازم. هیچ ابزار ویرایش کدی پیش از تایید شما استفاده نخواهد شد.

## ۲. ایجاد مستند راهنمای Git Backup
- **هدف:** توانمندسازی شما برای گرفتن بک‌آپ دستی و مدیریت سورس‌کدها.
- **تغییرات:** یک فایل متنی/مارک‌داون جدید با نام `git_backup_guide.md` در پوشه `Documents` ایجاد می‌شود. این فایل شامل پرکاربردترین دستورات Git (مثل وضعیت، کامیت، بازگردانی و تفاوت‌ها) به زبان ساده و کاربردی خواهد بود.

## ۳. قابلیت جستجوی متنی و چک‌باکس همزمان برای ستون تگ
- **هدف:** ارتقای قابلیت فیلتر در `DataTable` برای اعمال همزمان جستجوی متنی (تایپ بخشی از کلمه) و چک‌باکس (انتخاب تگ‌های موجود).
- **تغییرات پیشنهادی:**
  - **فرانت‌اند (`data-table.component.ts`):** 
    - اضافه کردن نوع فیلتر جدید به نام `checkbox_text`.
    - تغییر رابط کاربری فیلتر به گونه‌ای که هر دو المانِ "تکست باکس جستجو" و "دکمه بازشونده چک‌باکس‌ها" در کنار هم (یا روی هم) در بخش فیلتر نمایش داده شوند.
    - تفکیک کلید فیلتر برای این دو حالت (مثلاً `tag` برای چک‌باکس‌ها و `tag_search` برای ورودی متنی).
  - **فرانت‌اند (`dispatch.ts`):**
    - در تابع `loadItems`، کلید `tag_search` را به فیلتر جستجوی متنی (`tag__icontains` یا مشابه آن که در بک‌اند پشتیبانی می‌شود) و کلید `tag` را برای چک‌باکس‌ها مپ می‌کنیم.
    - همچنین کلید `tag` را به آرایه `inFields` اضافه می‌کنیم تا ارسال مقادیر چندگانه (چک‌باکس) به بک‌اند از طریق `tag__in` به درستی صورت گیرد.
  - **فرانت‌اند (`dispatch.html`):**
    - تغییر `filterType` ستون تگ از `checkbox` به `checkbox_text`.
