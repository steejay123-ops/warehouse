# گزارش پیاده‌سازی اتصال نسخه سیستم به دیتابیس

این سند حاوی گزارش تغییرات اعمال شده برای تبدیل شماره نسخه ثابت به نسخه داینامیک متصل به دیتابیس است.

## تغییرات انجام شده

### بک‌اند (Django)
| فایل تغییر یافته | توضیحات تغییر |
| :--- | :--- |
| `warehouses/services.py` | اضافه کردن کلید پیش‌فرض `'system_version': '1.0'` در `DEFAULT_SETTINGS`. |
| `warehouses/views.py` | ایجاد کلاس جدید `PublicConfigViewSet` با دسترسی `AllowAny` برای بازگرداندن اطلاعات پایه شامل `system_version` و `system_name`. |
| `config/urls.py` | اضافه کردن مسیر `/api/public/config/` که به `PublicConfigViewSet` متصل شده است. |

### فرانت‌اند (Angular)
| فایل تغییر یافته | توضیحات تغییر |
| :--- | :--- |
| `src/app/core/api/config-api.service.ts` | ایجاد سرویس جدید برای فراخوانی متد `GET` مربوط به پیکربندی عمومی بک‌اند. |
| `src/app/components/login/login.ts` | پیاده‌سازی `OnInit`، تزریق سرویس کانفیگ و فراخوانی متد جهت دریافت `system_version` در زمان بارگذاری کامپوننت. |
| `src/app/components/login/login.html` | جایگزینی متن ثابت با متغیر قالب‌بندی شده `{{ systemVersion }}`. |

## نتایج تست
- پروژه سمت فرانت‌اند بدون هیچ‌گونه اخطاری با موفقیت بیلد شد (`ng build`).
- پروژه بک‌اند نیز از نظر صحت سینتکس تأیید شد (`python manage.py check`).

اکنون کاربر در زمان ورود، شماره نسخه‌ای را می‌بیند که قابلیت تغییر از طریق مدیریت دیتابیس و تنظیمات بک‌اند را دارد.
