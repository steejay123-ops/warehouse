# طرح پیاده‌سازی سیستم نقش‌ها، دسترسی‌ها و تخصیص انبار (RBAC)

برای استانداردسازی سیستم و رفع نقص‌های امنیتی، طرح زیر را برای پیاده‌سازی پیشنهاد می‌کنم. این طرح دقیقاً با رابط کاربری (UI) که در فرانت‌اند طراحی شده هماهنگ است.

## اهداف طرح
۱. تعریف و اختصاص دسترسی‌های سیستمی (Permissions).
۲. تعریف گروه‌ها/نقش‌های سازمانی (Roles).
۳. قابلیت اختصاص هر کاربر به یک یا چند انبار (Warehouse Assignments).

---

## ⚠️ نکات نیازمند تایید (User Review Required)

۱. **استفاده از سیستم بومی جنگو برای نقش‌ها:**
در بک‌اند (Django)، ما می‌توانیم جداول کاملاً جدیدی برای Role بسازیم، یا از سیستم بسیار قدرتمند و آماده‌ی `Group` و `Permission` خود جنگو استفاده کنیم.
*پیشنهاد من:* استفاده از کلاس‌های `Group` به عنوان نقش (Role) است. این کار هم امنیت بالاتری دارد، هم با کتابخانه‌های استاندارد همخوانی دارد و سرعت توسعه را بالا می‌برد. آیا با این رویکرد موافقید؟

۲. **دسترسی به پروژه‌ها/انبارها:**
برای اینکه یک شمارشگر فقط انبارهای خودش را ببیند، فیلد `assigned_warehouses` (به صورت Many-to-Many) به جدول کاربر اضافه می‌شود. 
*سوال:* آیا اگر کاربری نقش "مدیر سیستم" داشته باشد، باید به طور پیش‌فرض به همه انبارها دسترسی داشته باشد، یا باز هم باید انبارها صریحاً به او اختصاص داده شوند؟ (من پیشنهاد می‌کنم دسترسی مدیر به همه انبارها اتوماتیک باشد).

## تغییرات فنی پیشنهادی (Proposed Changes)

### Backend (`accounts/models.py`)
- حذف فیلد ساده `role` از جدول `CustomUser`.
- اضافه کردن فیلد چند‌به‌چند: `assigned_warehouses = models.ManyToManyField(Warehouse)`

### Backend (`accounts/views.py` & `serializers.py`)
- ایجاد API برای خواندن و ایجاد نقش‌ها (`/api/roles/`) که به جدول `Group` متصل است.
- ایجاد API برای خواندن لیست کل دسترسی‌ها (`/api/permissions/`).
- ساخت خودکار دسترسی‌های سفارشیِ فرانت‌اند در دیتابیس طی یک پروسه اتوماتیک (مثل: `perm_wh_create`, `perm_rec_dispatch` و...).
- بروزرسانی `UserSerializer` برای ارسال و دریافت آرایه‌ای از ID نقش‌ها و ID انبارهای اختصاص‌یافته.

### Frontend (`warehouse-front`)
- اتصال فرم‌های موجود در فایل `users.ts` به APIهای جدید بک‌اند (به جای استفاده از دیتای Mock).
- آپدیت مدل `user.model.ts` برای هماهنگی با آرایه نقش‌ها و پروژه‌ها.

---
لطفاً موارد ذکر شده (مخصوصا بخش تاییدیه) را مطالعه کنید و در صورت موافقت، دستور شروع پیاده‌سازی را بفرمایید.
