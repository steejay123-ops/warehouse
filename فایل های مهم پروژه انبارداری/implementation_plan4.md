# 🎯 Implementation Plan — فاز ۴: بازنویسی صفحات داخلی

این فاز شامل بازنویسی صفحات اصلی (Projects، Dispatch، و Docs) برای استفاده از کامپوننت‌های مدرنی است که در فاز ۲ ساخته‌ایم.

## ⚠️ User Review Required
> [!IMPORTANT]
> در این فاز قصد داریم صفحات فعلی که کدهای کثیف، Modalهای هاردکد شده و ساختارهای قدیمی دارند را تمیز کنیم و به `ModalComponent`، `ConfirmDialog`، `StatusBadge` و `DataTable` متصل کنیم.
> 
> لطفاً طرح زیر را بررسی و تایید کنید.

## 🛠 Proposed Changes

### ۱. صفحه Projects (مدیریت انبارها)
- **جایگزینی Modalهای فعلی:** حذف ۳ مودال هاردکد شده (Add, Edit, Template) در HTML و استفاده از `<app-modal>` با انیمیشن‌های فاز ۲.
- **جایگزینی دیالوگ‌های تایید:** حذف alert/confirm ساده و اتصال به `ConfirmDialogService`.
- **بهینه‌سازی کدهای HTML:** تمیز کردن HTML صفحه Projects و استفاده از `StatusBadgeComponent` برای نمایش وضعیت پروژه‌ها.

### ۲. صفحه Dispatch (تخصیص و توزیع)
- **جایگزینی جدول:** حذف جدول سنتی هاردکد شده و استفاده از کامپوننت هوشمند `<app-data-table>` که قابلیت‌های Sort، Filter و Pagination را خودش هندل می‌کند.
- **استفاده از Pipes:** اعمال `StatusLabelPipe` برای ترجمه خودکار وضعیت‌ها.

### ۳. صفحه Docs (بارگذاری اکسل و Base Data)
- **جایگزینی File Upload:** حذف input file ساده و استفاده از کامپوننت جدید `<app-file-upload>` که دارای قابلیت Drag & Drop و ولیدیشن است.
- **ارتقا تجربه کاربری:** اضافه کردن انیمیشن‌های ورود (fade-in) به بخش‌های مختلف.

---

## 🧪 Verification Plan

### Manual Verification
1. کلیک روی **انبار جدید** در صفحه پروژه‌ها باید مودال جدید را باز کند.
2. بستن مودال با دکمه ESC کیبورد یا کلیک روی فضای تاریک بیرون آن باید عمل کند.
3. زدن دکمه **حذف کارگاه انبار** باید `ConfirmDialog` قرمز رنگ را نمایش دهد.
4. جدول صفحه Dispatch باید قابلیت مرتب‌سازی با کلیک روی عناوین ستون‌ها را داشته باشد.
5. صفحه Docs باید اجازه آپلود با Drag & Drop را بدهد.
